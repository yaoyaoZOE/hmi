# C++ 链接阶段（Link Stage）详解

链接（Linking）是 C++ 编译流程的最后一个阶段，它将多个编译单元（.o / .obj 文件）以及库文件“拼装”成最终的可执行文件或共享库。下面是链接器主要完成的工作。

## 整体编译流程回顾

```text
源文件 (.cpp)
    ↓ 预处理（Preprocessing）
    ↓ 编译（Compilation）
    ↓ 汇编（Assembly）
目标文件 (.o)
    ↓ 链接（Linking）  <- 我们关注的阶段
可执行文件 / 动态库
```

## 链接阶段的核心工作

### 1. 符号解析（Symbol Resolution）

- 每个 .o 文件都有一张符号表，记录它定义和引用的符号（函数名、全局变量等）。
- 链接器扫描所有目标文件，将未定义的符号引用与其他文件中的符号定义匹配起来。
- 如果某个符号找不到定义，会报链接错误：`undefined reference to 'xxx'`。
- 如果某个符号被定义了多次，会报链接错误：`multiple definition of 'xxx'`。

```cpp
// a.cpp 中引用了 foo()
extern void foo();
void bar() { foo(); }  // 符号引用

// b.cpp 中定义了 foo()
void foo() { /* ... */ }  // 符号定义
```

### 2. 重定位（Relocation）

- 编译阶段，每个 .o 文件中的地址都是相对地址（从 0 开始），是临时占位符。
- 链接器将所有目标文件合并后，为每个符号分配最终的虚拟地址。
- 然后回填所有引用该符号的地址占位符。

```text
编译后：CALL 0x00000000  <- 占位符
链接后：CALL 0x00401234  <- 真实地址
```

### 3. 合并段（Section Merging）

链接器将各个 .o 文件中相同类型的段（Section）合并到一起：

| 段名 | 内容 |
| --- | --- |
| .text | 所有目标文件的代码段合并 |
| .data | 已初始化的全局/静态变量 |
| .bss | 未初始化的全局/静态变量 |
| .rodata | 只读数据（字符串常量等） |

### 4. 库文件处理（Library Linking）

静态库（.a / .lib）：

- 链接器从静态库中提取需要的 .o 文件，直接合并进可执行文件。
- 最终文件体积较大，但运行时无依赖。

动态库（.so / .dll）：

- 链接器只记录库的名称和符号信息，不将库内容打包进去。
- 运行时由动态链接器（ld.so）负责加载。
- 最终文件体积小，但运行时需要库文件存在。

```bash
# 链接静态库
g++ main.o -L./lib -lmylib_static -o app

# 链接动态库
g++ main.o -L./lib -lmylib_shared -o app
```

### 5. 处理 C++ 特有内容

模板实例化去重：

- 同一个模板可能在多个 .o 文件中被实例化。
- 链接器负责去除重复的模板实例，只保留一份。

虚函数表（vtable）生成：

- 链接器负责最终确定并合并各类的 vtable。

全局对象的初始化顺序：

- 链接器收集所有翻译单元中全局/静态对象的构造函数。
- 生成初始化调用序列（通过 `.init_array` 或 `__attribute__((constructor))` 等机制）。

名称修饰（Name Mangling）：

- C++ 编译器会对函数名进行修饰（如 `_ZN3foo3barEi`）以支持重载。
- 链接器基于修饰后的名称进行符号匹配。

### 6. 生成最终输出文件

链接器根据链接脚本（Linker Script）的规则：

- 决定各段在内存中的布局。
- 设置程序入口点（通常是 `_start -> main`）。
- 生成 ELF（Linux）或 PE（Windows）格式的可执行文件。

## 完整流程示意

```text
[a.o] [b.o] [c.o]    [libstdc++.a]  [libm.so]
   \    |    /              |              |
    \   |   /         (提取需要的 .o)   (记录依赖)
     \  |  /
   [链接器 ld/lld]
         |
    符号解析 -> 重定位 -> 段合并 -> 生成输出
         |
   [可执行文件 app]
```

## 常见链接错误速查

| 错误信息 | 原因 |
| --- | --- |
| `undefined reference to 'xxx'` | 符号定义缺失，忘记链接某个库或 .o |
| `multiple definition of 'xxx'` | 同一符号被定义了多次 |
| `cannot find -lxxx` | 找不到指定的库文件 |
| `undefined reference to 'vtable for xxx'` | 虚函数未实现 |

总结来说，链接阶段的核心使命就是：把分散的零件（.o 文件和库）拼装成一个完整的、可以运行的程序。
